    Module uim.oop.patterns.interpreters.interfaces

    Interfaces

    NameDescription
    IBinaryExpression Binary expression has two operands (left and right).
    IContext Context holds information that is global to the interpreter. It typically stores variables and their values.
    IExpression AbstractExpression declares an interpret operation that all nodes in the abstract syntax tree must implement.
    IExpressionFactory Expression factory creates specific types of expressions.
    IExpressionOptimizer Expression optimizer transforms expressions into more efficient forms.
    IExpressionVisitor Visitor for expression trees (for operations like optimization, printing).
    IGrammar Grammar defines the language rules.
    IInterpreter Interpreter evaluates expressions in a specific language.
    ILiteralExpression Literal expression represents a constant value.
    INonterminalExpression Nonterminal expression represents composite nodes that combine other expressions using operators or rules.
    IParser Parser builds an abstract syntax tree from input text.
    ITerminalExpression Terminal expression represents leaf nodes in the abstract syntax tree. These are the basic building blocks that cannot be broken down further.
    IUnaryExpression Unary expression has one operand.
    IVariableExpression Variable expression represents a variable reference.
